# SEO Context — Ndauwo Safari Co.

## Project Overview
- **Site**: Ndauwo Safari Co. — Tanzania Luxury Safari Experiences
- **Stack**: Laravel 12 (API) + Vue 3 SPA (frontend)
- **Database**: MySQL (`ndauwo_db`)
- **Status**: Frontend and backend are separate projects — migration to unified Laravel project in progress

## SEO Goals
1. Server-rendered meta tags (title, description, OG, Twitter Card) for all public pages
2. JSON-LD structured data (Organization, WebSite, BreadcrumbList)
3. Proper canonical URLs
4. Hreflang tags for GTranslate multi-language support
5. XML Sitemaps (packages, destinations, posts)
6. robots.txt served correctly
7. Semantic HTML with proper heading hierarchy

## Current SEO State (Pre-Migration)
- Vue SPA renders entirely client-side
- Search engines see only bare `index.html` shell
- No server-injected meta tags
- No structured data
- No canonical URLs per page
- Sitemaps exist but are served via API routes only

## Target SEO State (Post-Migration)
- Blade entry point (`app.blade.php`) injects full SEO meta per route
- `SeoService` maps route patterns to optimized meta content
- JSON-LD structured data injected in `<head>`
- All meta is server-rendered before Vue mounts
- Clean URLs without hash fragments (vue-router history mode)
- Proper Open Graph images via settings/database
